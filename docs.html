Introduction

Astaraxia is built on the philosophy of transparency over magic. The system is designed to be auditable, minimal, and entirely source-based.

Note: Astral is written in pure POSIX shell. It assumes only a working /bin/sh and basic coreutils.

Astral Usage

Astral is the heart of the system. It handles building, staging, and installing recipes.

Basic Commands

# Install a package from source
astral -i <pkgname>

# Build a package without installing
astral -b <pkgname>

# List installed packages
astral -l

# Remove a package safely
astral -r <pkgname>

Writing Recipes

Recipes are simple directories containing build instructions. A valid recipe requires:

  • version: The version string.
  • build: A shell script for the compilation phase.
  • package: A shell script for the installation phase.

Example build script:

#!/bin/sh
./configure --prefix=/usr
make

Horizon Bootstrap

Horizon is the engine that drives the Astaraxia deployment process. It automates the transition from a host system to a standalone Astaraxia environment.

# Start the bootstrap
sudo astral -h --init
sudo astral -h --stage 1
